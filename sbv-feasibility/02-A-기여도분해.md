# A — "왜 떴나" 기여도 분해 (설계 + 실증)

> SBV의 'Program+Context 68.5% 지배' 통찰을 주식에 적용: 이상치를 **시장 / 섹터 / 종목고유**로 분해해 "테마가 같이 뜬 것"과 "진짜 고유 신호"를 구분. **예측 아님 — 설명/탐지.** 추가 데이터 0.

## 1. 가설 (라운드테이블)
주식판 '편성 ≠ 연출': 레이더가 띄우는 이상치 다수는 *그날 시장·섹터 레짐*이 만든 동반 움직임이지 종목 고유 신호가 아니다. → 분해해서 고유 이탈만 부각해야 정직하다.

## 2. 분해 모델 (프로토타입 — 가법 3분해)
```
ret_i = market + sector_i + specific_i
  market     = 그날 전체 50종목 평균 등락률
  sector_i   = (테마평균 등락률) − market
  specific_i = ret_i − 테마평균 등락률      ← 종목 고유 이탈
```
- 점유율: `shareSpecific = |specific| / (|market|+|sector|+|specific|)`
- 분류: `shareSpecific < 0.5 → 시장·섹터 주도`, 아니면 `종목 고유`.
- 데이터: `src/data/radar-frames.json`(per-stock 일일 등락률) + 종목 theme.

## 3. 실증 결과
스크립트 [prototype/decompose.mjs](prototype/decompose.mjs) 실행 → [prototype/result.md](prototype/result.md).

| 지표 | 값 |
|---|---|
| 분석 이상치(anomaly≥0.45) | **170건** |
| 시장·섹터 주도 | **85건 (50%)** |
| 종목 고유 주도 | 85건 (50%) |
| 평균 점유 — 시장+섹터 | **51%** vs 고유 49% |

**최신일 분해 예시**
| 종목 | 등락 | =시장 | +섹터 | +고유 | 주도 |
|---|--:|--:|--:|--:|:--:|
| 피에스케이 | +13.87% | −4.35 | +4.49 | +13.73 | 고유 |
| SK | −9.83% | −4.35 | −1.13 | −4.35 | 시장·섹터 |
| SK스퀘어 | −10.88% | −4.35 | −0.46 | −6.07 | 고유 |

→ 현 레이더 이상치의 **약 절반이 시장·섹터 동반**(고유성 낮음). 분해가 즉시 가능하고 유의미함을 데이터로 확인. 가설 **지지**.

## 4. 레이더 통합 방법 (구현 시)
- **Y축 교체**: raw `retPct` → **`specific`(시장·섹터 통제 후 고유 잔차)** σ 정규화. → 시장 따라 움직인 종목은 중앙으로 수렴, "혼자 튄" 종목만 가장자리.
- **블립 확장(하위호환 append)**: `[i,x,y,anomaly,relVol,retPct, retSpecific, ledBy, shareSpecific]`.
- **색 = ledBy**: 고유=한국식 등락색 풀채도(주목), 시장동반=회색(노이즈), 섹터동반=중채도+틸테두리.
- **크기/정렬 = `anomaly × shareSpecific`**: 고유성 높은 이상치를 더 크게/위로.
- **경보 패널**: "거래량 2.4배 · +5% **(시장 +3%, 고유 +2%)**" 식 분해 노출.
- 바뀌는 파일: `scripts/build-radar.mjs`, `src/lib/radarData.ts`, `src/components/StockRadar.tsx`, (`src/data/etf-stocks.json` 섹터매핑 보강 필요시).

## 5. 더 엄밀한 산식 (B 권고 — 구현 시 채택)
가법 3분해 → **횡단면 OLS 잔차**로 승격:
```
ret_{i,t} = α_t + β_t·ret_market,t + Σ_k γ_{k,t}·sector_dummy_{i,k} + ε_{i,t}
specific = ε_{i,t}
```
- 시장 프록시: KOSPI/KOSDAQ 지수(또는 유니버스 평균). look-ahead 차단: β는 t 이전 rolling 추정.

## 6. 한계·주의
- 프로토타입은 *전체 기간* 기준 분해(실증용). 실제 통합 시 **walk-forward·point-in-time** 강제(G1).
- theme `[0]`만 사용 → 섹터 분해 품질 좌우. 다중섹터·지수팩터 검토 필요.
- **이상 ≠ 매매신호.** 분해는 "어디서 왔나"를 설명할 뿐, 사라/팔라 아님.
